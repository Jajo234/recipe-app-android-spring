# Requisitos No Funcionales

Este documento describe los requisitos no funcionales del sistema Recipe App.

## Seguridad
RNF-01: El sistema deberá autenticar a los usuarios de forma segura.  
RNF-02: El sistema deberá restringir el acceso a funcionalidades según el estado de autenticación del usuario.  

## Rendimiento
RNF-03: El sistema deberá responder a las solicitudes en un tiempo máximo de 2 segundos bajo condiciones normales.  

## Arquitectura
RNF-04: El backend deberá estar desarrollado siguiendo una arquitectura en capas.  
RNF-05: La aplicación Android deberá implementar el patrón de arquitectura MVVM.  

## Mantenibilidad
RNF-06: El código deberá seguir buenas prácticas de programación y estar debidamente organizado.  
RNF-07: El backend deberá contar con documentación de la API mediante OpenAPI/Swagger.  

## Usabilidad
RNF-08: La interfaz de usuario deberá ser intuitiva y fácil de usar.  

## Escalabilidad
RNF-09: El sistema deberá permitir la incorporación de nuevas funcionalidades sin afectar las existentes.
